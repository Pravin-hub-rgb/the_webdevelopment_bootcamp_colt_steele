Strings

"STRINGS OF CHARACTERS"

Strings are another primitive type in JavaScript. They represent text, and must be wrapped in quotes.

                   
                   
                    let firstName = "Ziggy"; //Double quotes work
                    let msg = "Please do not feed the chimps!";
                    let animal = 'Dumbo Octopus'; //So do single quotes
                    let bad = "this is wrong'; //This DOES NOT work

It's fine to use either single or double quotes, just be consistent in your codebase.

                   

                    "He said 'hi' " //This can work

                    'He said "hi" ' //This can also

String Methods MDN reference

METHODS ARE BUILT-IN ACTIONS WE CAN PERFORM WITH INDIVIDUAL STRINGS

They help us do things like:

  • Searching within a string
  • Replacing part of a string
  • Changing the casing of a string

Syntax

thing.method()

Casing

toUpperCase()

                               
                               
                                let msg = 'I am king';
                                let yellMsg = msg.toUpperCase(); //'I AM KING'

toLowerCase()

                               
                               
                                let angry = 'LeAvE mE aLoNe!';
                                angry.toLowerCase(); //'leave me alone!'

So, the original are still same unless we update them.

Trim

It trims any void space at the beginning of a string or at the end of the string.

                               
                               
                                let greeting = '      leave me alone plz       ';
                                greeting.trim() //'leave me alone plz'

Strings Methods With Arguments

thing.method(arg)

Some methods accept arguments that modify their behavior.
Think of them as inputs that we can pass in. We pass these arguments inside of the parentheses.

indexOf

Returns the index within the calling string object of the first occurence of the specified value.

                               
                               
                                let tvShow = 'catdog';
                                tvShow.indexOf('cat');   //0
                                tvShow.indexOf('dog');   //3
                                tvShow.indexOf('z');   //-1 (not found)

slice

                               
                               
                                let str = 'supercalifragilisticexpialdocious';
                                str.slice(0,5);   //'super'
                                str.slice(5);   //'califragilisticexpialidocious'

replace

                               
                               
                                let annoyingLaugh = 'teehe so funny! teehe!';
                                annoyingLaugh.replace('teehe','haha');   //'haha so funny! teehee!'

Notice that it only replaces the first instance.

repeat

                               
                               
                                "lol".repeat(3);   //"lollollol"

Template Literals

Template literals are strings that allow embedded expressions, which will be evaluated and then turned into a resulting string.

                       
                       
                        `I counted ${3 + 4} sheep`;   //"I counted 7 sheep"
                        `Your total is $${44 * 5}`;   //"Your total is $220"

We use back-ticks not single quotes

Variables

VARIABLES ARE LIKE LABELS FOR VALUES

  • We can store a value and give it a name so that we can:
    • Refer back it later
    • use that value to do...stuff
    • Or change it later on.

BASIC SYNTAX

                   

                    let someName = value;

                    let year = 1985;

LET

LET is a keyword which comes before variable name while declaring variables.

                   
                   
                        let apple = 5;

                        // We can reassign value here
                        apple = 10;

                        // We cannot redeclare
                        let apple = 3; //ERROR

RECALL VALUES

                   
                   
                    let hens = 4;
                    let roosters = 2;
                    hens + roosters //6

UPDATING VARIABLES

                   
                   
                    let score = 0;
                    score //0
                    score = 5;
                    score //5
                    score = score + 5;
                    score //10

                    // shortcut syntax
                    score += 5;

                    // Increment by one
                    score++;

                    // Decrement by one
                    score--;

CONST

const works just like let, except you CANNOT change,reassign or update the value

                   
                   
                    const hens = 4;
                    hens = 20; //ERROR

WHY USE CONST?

Used to store constant value which should be not changed in program

                       
                       
                        const pi = 3.14159;
                        const daysInWeek = 7;
                        const minHeightForRide = 60;

We use const in arrays because the variable is storing the address of an array. So, the values can change as long as the reference remains the same.

VAR

THE OLD VARIABLE KEYWORD

Before let & const, var was the only way of declaring variables. These days, there isn't really a reason to use it.

Variables can change types

                   
                   
                    let numPuppies = 23; //Number
                    numPuppies = false; //Now a Boolean
                    numPuppies = 100; //Back to Number!

It doesn't really make sense to change from a number to a boolean here, but we can!

In TypeScript we cannot change the variable types.

Variable Naming & Conventions

There are some rules in JavaScript that we should follow for variable naming

                   
                   
                        // No space in name
                        let hello world; //ERROR

                        // No digit at the beginning.
                        let 1game; //ERROR

                        // Use camel case
                        let camelCaseVersion = true;
