TERMINAL

Text based prompt where I can interact with machine through text commands.

Why do we need to know Terminal Commands

Speed!

Develop Faster
The terminal takes some getting used to, but it can be MUCH faster than using GUI(graphical user interface)

Access

With Great Power
The terminal provides a "mainline" into the heart of our computer, giving us access to areas we normally don't interact.

Tools!

Many of the tools we need are installed and used via the command line. We don't have much of a choice!

Terminology

Terminal

A text-based interface to your computer.
originally a physical object, but not we use software terminals.
E.g. → hyper.

Shell

The program running on the terminal
Each program have different commands.
E.g. → Bash

Bash

One of the most popular shells (default on mac)

Essential Commands

LS (list)

Use ls to list the contents of your current directory

PWD (Print Working Directory)

Prints the path to the working directory (Where you currently are)

CD (Change Directory)

Use cd to change and move between folders

                            

                                // cd directory_name
                                // Use cd .. to "back up" one directory
                                cd ..

Relative Vs. Absolute Paths

Absolute paths we can use from any directory while relative paths are always relative to current directory.

                            

                                cd /users/xyz/abc absolute path

                                cd / '/' represents root directory.Everything that we want to work with is somewhere nesting inside it.
                                
                                cd ~ '~' represents home directory

mkdir (make directory)

mkdir will create a new directory (or directories)

                            

                                // Making two directories
                                mkdir first second 
                                // This make two directories named 'first' and 'second'

man (mannual)

It doesn't impact the file system but it gives information (help command)

man ls Gives all information about ls command
To quit press 'q'

flags

These are options which enables how certain command should work

                                

                                    // ls -l give long information of each file in that particular directory.
                                    // ls -a shows directory entries whose names begin with a dot (.). (hidden files)
                                    
                                    // combining multiple flags
                                    ls -la

touch

Use touch to create a file (or multiple)

                            

                                touch purple.txt bubbs.html

rm

rem will delete a file or files
It permanently removes them!

                            

                                rm app.cs index.html // both these file are gone

                                // now to delete directory which are empty we use rmdir
                                rmdir empty_directory/

rm -rf

use rm -rf to delete directory (r = recursive, f = force)

recursive tells it to remove the subdirectories too

                            

                                rm -rf Plants/
