What is NODE?

"A JavaScript Runtime"

Until recently, we could only run JavaScript code in a web browser. Node is a javaScript runtime that executes code outside of the browser.

We can use the same JavaScript syntax we know and love to write server-side code, instead of relying on other languages like Python or Ruby.

What is Node used for?

  • Web servers → writing server side logic for a full stack application.
  • Command Line Tools
  • Native Apps
    • VS code is written in javaScript using node
    • Electron
  • Video Games
    • cross-code
  • Drone Software
  • A whole lot more!

Node REPL

When we type node in terminal we enter in node REPL(Read Evaluate Print Loop)
We can run JS here too

Node JS Vs. Client-side JS

Not included in Node

Because Node does not run in a browser, we don't have access to all the browser "stuff".
The window, document, and DOM API's are not a thing in Node!

New stuff in Node!

Node comes with a bunch of built-in modules that don't exist in browser.
These modules help us do things like interact with the operating system and files/folders

As in browser 'window' is top level object and in Node 'global' is top level object where everything live inside of.

Process & Argv

These stuff are not important for web development but if you want to make some applications than there it might help

Process

The process object is a global that provies information about, and control over, the current Node.js process. It is always available to Node.js applications without using require(). It can also be explicitly accessed using require(). 

                        

                            // inside node
                            process.version //gives version of node
                            process.release //info about release of node
                            process.cwd() //current working directory

process.argv

The process.argv property returns an array containing line arguments passed when the Node.js process was launched. The first element will be process.execPath. The second element will be the path to the JS file being executed. The remaining elements will be any additional command line arguments. 

                                

                                    //app.js

                                    console.log(process.argv);

Make directory and name it as passed argument

                        

                            //Terminal

                            node app.js chickens //directory named 'chickens' will be created                                    
                        
                    
                        

                            //app.js
                                    
                            //Make directory and name it as passed argument
                            const fs = require('fs');
                            let folderName = process.argv[2] || 'Project' //If arguments is not passed then 'Project' will be the default name of directory
                            fs.mkdirSync(folderName);
