HTML

Definition: Html is the underlying structure of a webpage.
Full form: Hypertext Markup Language

HTML 5

HTML 5 is the latest evolution of the standard that defines HTML. The term represents two different concepts. It is a new version of the language HTML with new elements, attributes, and behaviors and a larger set of technologies that allow the building of more diverse and powerful Web sites and applications.

Html Skeleton

  • The vs code shortcut for loading skeleton is "! + tab".
  • We write our HTML in a standard "skeleton."
  • <!DOCTYPE html> → tells the version of HTML that we are using.
  • The doctype for HTML5 is very simple. To indicate that your HTML content uses HTML5, simply use <!DOCTYPE html>
  • The browsers that don't presently support HTML 5 will enter "standards mode," which means that they'll interpret the long-established parts of HTML in an HTML5-compliant way while ignoring the new features of HTML5.
  • <html>
    • The HTML root element (<html>) represents the root of an HTML document. All other elements must be descendants of this element.
  • <head> → The document metadata (header) element
    • contains machine-readable information (metadata) about the document, like its title, scripts, and style sheets.
  • META TAGS
    • provides information to the browser about our websites.
    • Meta tags go to the head of the html boilerplate.
      1. <meta charset=”UTF-8”>
      2. <meta name=”description” content =”This page is a webpage to learn html”>
      3. <meta name=”keywords” content=”html5, blah, blah"”> → So, when these keywords are searched, the search engine will show my website in the search results.
      4. <meta name=”author” content=”Pravin”>
      5. <meta name=”viewport” content=”width=device-width, initial-scale=1.0”> → To be responsive on any device with different screen sizes.
  • <body>
    • represents the content of an HTML document. There can be only one element in a document.

Semantic Markup

  • Semantic: relating to meaning
  • These are new elements.
  • HTML should be coded to represent the data that will be populated and not based on its default presentation styling. Presentation (how it should look) is the sole responsibility of CSS.
  • Some of the benefits of writing semantic markup are as follows:
    • Search engines will consider its contents as important keywords to influence the page’s search rankings (see SEO).
    • Screen readers can use it as a signpost to help visually impaired users navigate a page.
    • Finding blocks of meaningful code is significantly easier than searching through endless divs with or without semantic or namespaced classes.
    • Suggests to the developer the type of data that will be populated.
    • Semantic naming mirrors proper custom element/component naming.
  • Instead of DIVs, use more specific elements like:
    • <section>
    • <article>
    • <nav>
    • <main>
    • <header>
    • <footer>
    • <aside>
    • <summary>
    • <details>
  • Why use these?
    • 1-maintainability
    • 2-accessibility
    • 3-SEO (search engine optimization)
      • code reading code.
  • A brief Explanation of each
    • ☆<nav>☆ element represents a section of a page whose purpose is to provide navigation links, either within the current document or to other documents. Common examples of navigation sections are menus, tables of contents and indexes.
    • ☆<header>☆ represents introductory content, typically a group of introduction or navigation aids, it may contain some heading elements but also a logo, a search form, an author name, and other elements.
    • ☆<footer>☆ represents introductory content, typically a group of introductions or navigation aids; it may contain some heading elements but also a logo, a search form, an author name, and other elements.
    • ☆<article>☆ represents a self-contained composition in a document, page, application or site which is intended to be independently distribute or reusable. Example include a form post, a magazine or a blog entry.
    • ☆<section>☆ represents the parts of an article.
    • ☆<aside>☆ represents a portion of a document whose content is only indirectly related to the document's main content. Asides are frequently presented as sidebars or call-out boxes.
    • ☆<main>☆ represents the dominant content of the <body> of a document. The main content area consists of content that is directly related to or expands upon the central topic of document, or the central functionality of an application. it should be the core content which is unique to the web.
    • ☆<time>☆ The HTML <time> element represents a specific period in time. It may include the datetime attribute to translate dates into machine-readable format, allowing for better search engine results, or custom features such as reminders.
    • ☆<figure>☆ The HTML <figure> (Figure with optional caption) element represents self-contained content, potentially with an optional caption, which is specified using the (<figcaption>) element. The figure, its caption, and its contents are referenced as a single unit.
