A New EJS Tool For Layouts

Before we used to take help of Partials but now we will use a package 'ejs-mate'. It allow us to use fun usefull functionality that is something called Layout. ejs-mate NPM docs! ejs-mate Github repo! 

                        

                            //layouts/boilerplate.ejs
                            <!DOCTYPE html>
                            <html lang="en">
                            
                            <head>
                                <meta charset="UTF-8">
                                <meta http-equiv="X-UA-Compatible" content="IE=edge">
                                <meta name="viewport" content="width=device-width, initial-scale=1.0">
                                <title>Boiler palte!!!</title>
                            </head>
                            
                            <body>
                                <h1>Before</h1>
                                <%- body %>
                                <!- When called anywhere inside a template, requests that the output
                                    of the current template be passed to the given view as the body local. Use this to specify
                                    layouts from within your template, which is recommended with Express 3.0, since the
                                    app-level layout functionality has been removed. -->
                                <h1>After</h1>
                            </body>
                            </html>
                    
                    
                        

                        //So this is one of many engines that are used to run or parse and basically make sense for ejs 
                        const ejsMate = require('ejs-mate');
                        //here we are telling Express to ejsMate engine instead of relying on the default one.
                        app.engine('ejs', ejsMate);
                        
                    
                        

                         <% layout('layouts/boilerplate') %>
                         <h1>All campgrounds</h1>
                         <div>
                            <a href="/campgrounds/new">Add Campgrounds</a>
                         </div>
                         <ul>
                            <% for (let campground of campgrounds) {%>
                            <li>
                                 <a href="/campgrounds/ <%=campground._id%>"> <%=campground.title%> </a>
                            </li>
                            <% } %>
                         </ul>
                        //now we are successfully injecting our content in this layout.
                        //now update all other template to use this
